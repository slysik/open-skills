How these skills are optimized

Every skill was rewritten to a measured standard — small context footprint, deterministic auth, and zero runtime dependencies.

🪶

Thin-router architecture

SKILL.md is a router, not a manual — a concept map plus a "load which sub-doc" table. Average 76 lines, hard cap 150. Depth lives in references/.

🎯

Progressive disclosure

Sub-docs load into context only when the task needs them. A typical invocation reads tens of lines instead of thousands — large suite, small token bill.

⛔

CLI-first · MCP-never

Vendor CLI / REST is the only required path. No MCP servers to install, register, or keep alive — nothing to babysit between sessions.

🔐

Connection contract

Every auth doc uses the same four headings — Interactive · Service principal · Verify · Troubleshoot — so the agent never guesses how to authenticate.

✅

Preservation-tested trims

Routers were shrunk by moving body to references/ verbatim; an automated check proves no command or code line was lost.

🖥️

No UI, fully reproducible

Everything runs from a shell — scriptable, diffable, CI-able. Skills are versioned folders; the installer just copies them where your agent looks.
